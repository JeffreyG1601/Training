CSS Flexbox (display: flex)

Flexbox is a CSS layout model that makes it easy to design flexible and efficient layouts, align items, and distribute space within a container, even when their size is unknown or dynamic.

How Flexbox Works

  • Set display: flex on a container to make its children flexible items.
  • Use properties like justify-content, align-items, and flex-direction to control alignment and layout.

Key Concepts of Flux

        
    
            
  • Actions: Plain JavaScript objects that describe what happened. They are sent to the dispatcher to signal that something has changed.
    • 
            
  • Dispatcher: A central hub that manages all data flow in a Flux application. It receives actions and dispatches them to the stores.
    • 
            
  • Stores: Containers for application state and logic. Stores register with the dispatcher and update themselves in response to actions.
    • 
            
  • Views: React components (or other UI components) that listen to stores for changes and re-render accordingly.
    • 
        


        

            
            
Flux unidirectional data flow

        


        
How Flux Works: The Data Flow

        
    
            
  1. Action is triggered: The user interacts with the view (e.g., clicks a button), which creates an Action.
    2. 
            
  2. Dispatcher dispatches: The Action is sent to the Dispatcher, which forwards it to all registered Stores.
    3. 
            
  3. Stores update state: Each Store checks the action type and updates its state if necessary.
    4. 
            
  4. View updates: The Store emits a change event. The View listens for this event and re-renders itself with the new data.

    Common Flexbox Properties
    
        
      
            
    • display: flex — enables flex context for all its children
      • 
            
    • flex-direction — row (default), column, row-reverse, column-reverse
      • 
            
    • justify-content — aligns items horizontally (start, end, center, space-between, space-around, space-evenly)
      • 
            
    • align-items — aligns items vertically (start, end, center, stretch, baseline)
      • 
            
    • flex-wrap — controls whether items wrap onto multiple lines
      • 
            
    • flex — shorthand for flex-grow, flex-shrink, and flex-basis
      • 
        
    

        
    Summary
    
        
    Flexbox is a powerful tool for building responsive layouts in CSS. It simplifies alignment, spacing, and distribution of elements within a container.
    
        


        
Why Use Flux?

        
    
            
  • Predictable state management with a clear data flow.
    • 
            
  • Easy to debug and test, since data only flows in one direction.
    • 
            
  • Decouples state and UI, making code more modular and maintainable.
    • 
        


        
Flux vs. MVC

        
Unlike traditional MVC (Model-View-Controller), where data can flow in multiple directions, Flux enforces a strict one-way data flow. This reduces complexity and makes the application easier to reason about.
